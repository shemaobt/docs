# Content Governance

## Writing standard

- Prefer short sections with clear headings.
- Keep content factual and implementation-focused.
- Use MDX where interactive or rich formatting improves clarity.
- Add links to source repositories and RFCs.

## Update cadence

- Update system pages when behavior, interfaces, or deployment changes.
- Review cross-system architecture pages during major releases.
- Archive or mark stale docs explicitly.

## Definition of done for docs

- Scope and ownership are clear.
- Architecture and runtime details are present.
- Integrations and dependencies are listed.
- Known limitations and next steps are documented.
