# Documentation Templates

## System page template

Use this template for new entries under `Systems`.

```md
# <system-name>

## Purpose
## Core flows
## Architecture
## Runtime and deployment
## Integrations
## Related RFCs
## Roadmap and open questions
```

## RFC template checklist

- Context and motivation
- Problem statement
- Constraints
- Options and trade-offs
- Recommended option
- Risks and mitigations
- Rollout plan
- Success criteria

## MDX guidance

- Prefer Markdown for static text.
- Use MDX when embedding diagrams, callouts, or reusable components.
- Keep pages readable in raw Markdown form when possible.
